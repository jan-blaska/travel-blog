# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev      # start dev server with Turbopack
npm run build    # production build
npm run lint     # ESLint
```

No test suite is configured.

## Architecture

**Stack**: Next.js 16 App Router, TypeScript, Tailwind CSS v4, `next-intl` (i18n), `next-mdx-remote` (MDX content), `next-themes` (dark mode), Leaflet (interactive map).

### Routing & i18n

All routes live under `src/app/[locale]/` — every page receives a `locale` param (`en` or `cs`, default `en`). Each page must call `setRequestLocale(locale)` at the top. Translation messages are split by page into `messages/{locale}/*.json` and merged in `src/i18n/request.ts`. Use the typed navigation helpers from `src/i18n/navigation.ts` (wraps `next-intl`) instead of plain Next.js `Link`/`useRouter`.

### Content system

Travel articles are MDX files at `src/content/{locale}/{continent}/{country}/{slug}.mdx`. Country overview pages use `index.mdx` in the same directory. `generateStaticParams` in the page files walks this filesystem tree at build time — no manual registration needed for new articles.

The destinations list in `src/app/[locale]/adventures/destinations.ts` (and the same for `travel-tips/`) controls which countries appear in the navigation list. **When adding a new country, update this file and add the translation key to all `messages/{locale}/adventures.json` files.**

### MDX components

These components are available inside `.mdx` files without importing them — they are injected by `compileMDX`:

| Component | Use |
|---|---|
| `MainHeader` | Section heading with green underline |
| `ImageNextToText` | Image beside a text block (supports `reverse` prop) |
| `ImageNextToImage` | Two images side by side |
| `ImageSlider` | Auto-advancing carousel (client component) |
| `ImageSliderWide` | Full-width carousel |
| `CardLink` | Card with image, title, description linking to a route |

### Styling

Tailwind CSS v4 is used. Dark mode is toggled by adding the `dark` class to `<html>` (configured in `globals.css` with `@custom-variant dark`). CSS variables for colors (`--green`, `--orange`, `--background`, etc.) are defined in `globals.css` and referenced as `text-(--green)` etc.

Custom fonts are loaded in the root layout and exposed as Tailwind utilities: `.font-montserrat`, `.font-comforter`, `.font-barlow-condensed`, `.font-cinzel`.

### Leaflet map

The `Map` component (`src/app/[locale]/components/Map.tsx`) must be imported with `dynamic(..., { ssr: false })` because Leaflet requires browser APIs.
